# Заметки Докер


## Определение
**Docker** — платформа для разработки, доставки и запуска приложений в контейнерах. Контейнеры представляют собой изолированные среды, содержащие приложение со всеми его зависимостями (библиотеки, конфигурации и т.д.), что обеспечивает единообразие работы на любой инфраструктуре.

## История появления и развития
- **2013 год** — основатель Соломон Хайкс представляет Docker как проект с открытым исходным кодом.
- **2014 год** — выход версии 1.0, быстрое распространение технологии благодаря удобству использования.
- **2015–2017** — рост экосистемы, появление инструментов оркестрации (Kubernetes, Docker Swarm).
- **С 2018 года** — стабилизация, интеграция в CI/CD, облачные платформы (AWS, Azure, GCP).

## Современное состояние
- **Стандарт** для контейнеризации приложений и микросервисной архитектуры.
- **Активно используется** в DevOps, облачных развертываниях, автоматизированных pipelines.
- **Поддержка** основных ОС (Linux, Windows, macOS).
- **Развитие** в сторону безопасности, управления жизненным циклом и интеграции с Kubernetes.

## Docker Hub и готовые образы
**Docker Hub** — облачный реестр (registry) для хранения и распространения Docker-образов.

**Возможности:**
- Публичные и приватные репозитории.
- Официальные образы (OS, языки программирования, СУБД).
- Автоматические сборки (Auto-build) из GitHub/GitLab.
- Интеграция с инструментами CI/CD.

**Примеры готовых образов:**
- `nginx`, `postgres`, `python:3.9-alpine`, `node:latest`

## Dockerfile

**Пример Dockerfile:**
```dockerfile
FROM python:3.9-alpine
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python", "app.py"]
